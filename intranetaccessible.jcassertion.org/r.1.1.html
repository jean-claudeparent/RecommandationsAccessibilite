Règle 1.1 Les équivalents textuels

1.1.1 Contenu non textuel : tout contenu non-textuel présenté à l'utilisateur a un équivalent textuel qui remplit une fonction équivalente.

Comment tester

La fonction "Audit / Accessibility" des version récente de chrome détecte ce problème

1.1.1.i1 Éléminer les situations de recours à un seul sens

Éliminer les situation où on recours à des éléments s'adressant à un seul sens. Par exemple:

  • Passer un test oz donner l'équivalent texte donnerait la réponse, par exemple reconnaître un logo.
  • Des éléments de site orientées pour faire ressentir une expériencesensorielle par un sens précis. Par exemple une grosse animation d'un paysage.
  • UnCAPTCHA (test de turing) reposant sur un seul sens.Par exemple demande de sélectionner toutes les images représentant un chien parmi 10 images. Un intranet devrait utiliser un autre moyen pour s'assurer de l'identité de ses utilisateur.

Comment tester

La décision d'éliminer ces situations devrait se prendre à la conception du site.

1.1.1.i2 Rendre invisibles les éléments purement décoratifs

Les éléments purement décoratifs devraiemt être rendus invisibles aux technologies d'assistances. Par exemple une image qui est juste un dégradé de courleur pour faire joli devrait avoir une chaîne de caractère vide comme texte alternatif.

Comment tester

Essais manuel du site avec NVDA. Les validateurs automatiques ne sont pas en mesure de déterminer quelles sont les images purement décorative.
