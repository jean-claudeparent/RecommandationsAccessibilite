Les outils pour tester l'accessibilité

Stratégie d'essais

Comme l'accessibilité repose sur l'interaction de plusieurs éléments il faut absolument tester. Le respect théorique des règles n'est pas suffisant. Les essais se font en deux partie:

  • Les essais d'analyse du code des pages. Ceci est mieux accompli par un logiciel d'analyse statique des pages.
  • Les essais du résultat global de l'accessibilité. Ceci est fait par une personne jouant le rôle d'une personne handicapée en utilisamt des technologies d'assistance. Ces essais sont aussi appelés essais manuels par opposition aux essais automatisés.

Il est à noter que c'est un processus itératif. Un changement fait suite aux essais globaux devrait être retester avec l'outil automatisé.

Analyse statique du code

Il s'agit souvent d'un logiciel qui analyse le code html des pages web. Cet essais peut être totalement ou partiellement automatisé. On parles d'analyse statique du code parce que le logiciel regarde une page à un moment donnée. Le logiciel ne peut pas trouver les problèmes qui apparaissent uniquement quand l'utilisateur travaille avec la page.

Lighthouse

Lighthouse est un logiciel qui peut faire des audit de page web. Il y a nottamment un audit d'accessibilité qui est assez bon. LightHouse peut être accédé par les outils pour les développeur de chrome. Les outils développeurs sont dans le menu de google chrome car ils sont pré installées sur chrome pour ordinateur de bureau. Lighthouse peut aussi être utilisé d'autres façon comme un validateur en ligne de commande. Il peut aussi être ajouté dans un scénario d'intégration continue.

Avantages

  • Gratuit et déjà pré installé dans les versions de google chrome.
  • Facile à installer pour une utilisation en ligne de commande.
  • Supporte une variété de cas d'utilisation.
  • Donne d'excellent explication et suggestions pour corriger ce qui ne va pas.
  • Sécuritaire lorsque installé sur votre ordinateur ou vo serveurs, tout se passe sur votre réseau interne.
  • Avec chrome on peut auditer des pages qui requiert une authentification.
  • Donne des suggestions de test à faire manuellement.

Désavantages

  • Disponible uniquement en anglais.
  • N'indique pas les numéros de critères de succès du WAG 2.1 qui ne passent pas. Il peut aussi sortir des erreurs d'accesibilité dépassant le niveau A de conformité.

Essais globaux d'accessibilité

Ces essais aussi appelés essais manuels sont la braie preuve que l'accessibilité fonctionne. En effet une analyse statique par logiciel peut ne rien trouver comme problème d'accessibilité et que cependant la page soit totalement innaccessible. Ces essais se font en utilisant des technologie d'assistance réelle. Les essais peuvent être fait par une personne qui est rééellement handicapée, mais cette personne doit maîtriser un vocabulaire de l'accessibilité permettan de se rattacher aux règles du EAG.

Navigateur standard avec son fermé

Pour les essais concernant les limitations d'audition, simplement fermez le son de votre ordinateur.

Lecteur d'écran NVDA

Pour jouer le rôle d'une personne ayant des limitations au niveau de la vision, autant utiliser un vrai lecteur d'écran. Un lecteur d'écran est un logiciel qui lit l'information affichée sur l'ordinateur de l'utilisateur et qui la prononce avec une synthèse vocale. Le logiciel peut aussi envoyer l'information sur un afficheur braille.

NVDA (non visual desktop access) est un lecteur d'écran suffisamment performant pour être vraiment utilisé par des personnes handicapées. Ce logiciel fonctionne uniquement sur windows et est un logiciel libre.

Avantages

  • Gratuit.
  • Produit mature et représentatif des technologies d'assistance.
  • Disponible en français
  • Il y a une version portable ne nécessitant pas d'installer des composantes sur l'ordinateur. On peut donc mettre NVDA sur une clé usb et changer facilement d'ordinateur en exécutant NVDA à partir de la clé USB.

    • Désavantages

    • Il faut parfois chercher un peu dans la con figuration pour trouver une voix que vous comprenez bien.
    • Il faut prendre le temps d'apprendre comment l'utiliser. De même plusieurs personnes doivent écouter la boix de synthèse pendant un certain temps avant de la comprendre.

Note : Vous êtes productif avec NVDA quand vous pouvez utiliser un site accessible en fermant l'écran de votre ordinateur.
