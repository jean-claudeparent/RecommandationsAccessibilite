Les outils pour tester l'accessibilité

Stratégie d'essais

Comme l'accessibilité reoise sur l'interaction de plusieurs éléments il faut absolument tester. Les essais se font en deux partie:

  • Les essais d'analyse du code des pages. Ceci est mieux accompli par un logiciel d'analyse statique des pages.
  • Les essais du résultat global de l'accessibilité. Ceci est fait par une personne jouant le rôle d'une personne handicapée en utilisamt des technologies d'assistance.

Il est à noter que c'est un processus itératif. Un changement fait suite aux essais globaux devrait être retester avec l'outil automatisé.

Analyse statique du code

On parles d'analyse statique du code oarce que le logiciel regarde une page à un moment données. Le logiciel ne peut pas trouver les problèmes qui apparaissebt uniquement quand l'utilisateur travaille avec la oage. Lighthouse

Lighthouse est un logiciel qui peut faire des audit de pahe web. Il y a nottamment un audit d'accessibilité qui est assez bon. LightHouse peut petre accédé par les outils pour les développeur de chrome Il peut aussi être utilisé d'autres façon comme un validateur en ligne de commande. Il peut aussi être ajouté dans un scénario d'intégration continue.

Avantages

  • Déjà pré installé dans les versions de google chrome.
  • Déjà pré installé dans les versions de google chrome.
  • Facile à installer pour une utilisation en ligne de commande.
  • Supporte une variété de cas d'utilisation.
  • Donne d'excellent explication et suhhestions pour corriger ce qui ne va pas.
  • Sécuritaire, tout se passe sur votre matériel.
  • Avec chrome on peut auditer des pages qui requiert une authentification.
  • Donne des suggestions de test à faire manuellement.

Désavantages

  • Disponible uniquement en anglais.
  • N'indique pas les numéros de critères de succès du WAG 2.1 qui ne passent pas.
